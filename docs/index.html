Privacy-First Time Tracker

Offline time tracking.
Your data never leaves your Mac.

Unlike cloud-based time trackers, Pulse runs 100% offline. No accounts, no subscriptions, no data collection. The most private time tracking app for macOS — open source and auditable.

No Network Dependencies

The only dependencies are SQLite and a CLI parser. Nothing phones home.

// Package.swift — complete dependency list dependencies: [ .package(url: "SQLite.swift", from: "0.15.3"), .package(url: "swift-argument-parser", exact: "1.3.1"), ] // That's it. No analytics. No tracking SDKs.

Local SQLite Storage

All data lives in a single file on your machine. You own it completely.

// ActivityStore.swift — data path let appSupport = FileManager.default.urls( for: .applicationSupportDirectory, in: .userDomainMask ).first!.appendingPathComponent("ActivityTracker") path = appSupport.appendingPathComponent("activity.db") // ~/Library/Application Support/ActivityTracker/

Smart Idle Detection

Never tracks when you're away. Auto-pauses after 10 minutes of inactivity.

// IdleDetector.swift — system idle check let idleSeconds = CGEventSource.secondsSinceLastEventType( .combinedSessionState, eventType: CGEventType(rawValue: ~0)! ) let nowIdle = idleSeconds >= 600 // 10 minutes

FAQ

Frequently asked
questions.

How is Pulse different from manual time trackers?

Manual time trackers require you to start and stop timers for each task. If you forget, your data is inaccurate. Pulse tracks your time automatically by monitoring which app and window is active on your Mac. No timers, no buttons, no forgetting. It also auto-categorizes your work into projects using AI pattern detection — no manual tagging needed.

Can Pulse replace manual timesheet logging?

Yes. Instead of manually logging hours at the end of each day, Pulse captures this data automatically — every minute spent in VS Code, Terminal, Figma, or your browser is recorded with the exact window title. You can create brand/project hierarchies and Pulse will auto-assign activities using classification rules.

Is Pulse free? What's the catch?

Pulse is 100% free and open source under the MIT license. There's no catch, no premium tier, no data harvesting. It runs entirely on your Mac with no server, no account, and no subscription. The source code is fully auditable on GitHub.

Does Pulse work offline?

Yes, Pulse works 100% offline. It uses a local SQLite database stored on your Mac. No internet connection is required for tracking, reporting, or project management. Your data never leaves your machine — making it the most private time tracking app available for macOS.

What apps does Pulse track?

Pulse tracks every macOS application: VS Code, Figma, Safari, Chrome, Terminal, Slack, Notion, and more. It also separates browser tabs — YouTube, GitHub, ChatGPT, and other sites are tracked as individual apps. Terminal background processes (builds, deploys, npm scripts) are tracked with project names extracted from window titles.

How does automatic project classification work?

Pulse uses 7 rule types to classify activities: URL domain, URL path, window title, terminal folder, Figma file name, bundle ID, and page title. You can create rules manually, or let the AI pattern detector scan your unassigned activities and suggest rules automatically. Once set, rules auto-classify all future activities — no manual tagging needed.

Is Pulse good for freelancers?

Pulse is built for freelancers, contractors, and agencies who need accurate time tracking for client billing. It automatically tracks which client project you're working on, generates detailed reports showing hours per project, and separates billable work from personal browsing. No more guessing how long a task took — every minute is captured.

Why choose Pulse over manual time tracking tools?

Manual time tracking tools require discipline — starting timers, stopping them, tagging projects. Pulse runs independently in your menu bar and captures everything automatically. You get accurate time data without the manual effort, without monthly costs, and without any cloud dependency.
